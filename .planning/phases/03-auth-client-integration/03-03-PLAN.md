  <action>
  Create the apps/mobile/app/ directory and root _layout.tsx. This file is the Expo Router entry point — it replaces App.tsx as the top-level component.

  Create apps/mobile/app/_layout.tsx:
  ```typescript
  import { Stack } from 'expo-router'
  import { PaperProvider } from 'react-native-paper'
  import { SafeAreaProvider } from 'react-native-safe-area-context'

  export default function RootLayout() {
    return (
      <SafeAreaProvider>
        <PaperProvider>
          <Stack screenOptions={{ headerShown: false }} />
        </PaperProvider>
      </SafeAreaProvider>
    )
  }
  ```

  Key constraints:
  - NO `'use client'` directive — Expo/React Native has no Server Components
  - `headerShown: false` — auth screens render their own headings per UI-SPEC; no native nav header
  - PaperProvider provides the MD3 theme used by React Native Paper components (plan 03-05)
  - SafeAreaProvider is required by react-native-safe-area-context (peer dep of RN Paper)
  - AuthProvider (mobile AuthContext) is NOT added here — that's plan 03-05 which modifies this file

  Note: plans/03-05 will modify this file to add AuthProvider, so 03-05 must declare `apps/mobile/app/_layout.tsx` in its files_modified and depends_on 03-03.
