# Plan 13-04 — Mobile UI: Client Identity on Job Detail

## Goal

Add the `getUser` API helper to the mobile lib, create a mobile `AvatarInitials` component, and add the inline client identity section to the mobile job detail screen. Runs in parallel with Plan 13-03 (both depend on 13-02).

## Must-Haves

- `apps/mobile/lib/api.ts` exports `getUser(token: string, userId: string): Promise<PublicUserDto>` following the exact signature of `getJob`.
- `PublicUserDto` is imported from `@local/types` in the mobile API file.
- `apps/mobile/components/AvatarInitials.tsx` exists; accepts `{ name, email, avatarUrl, size: number }` props; uses `Avatar.Text` / `Avatar.Image` from `react-native-paper`.
- `apps/mobile/app/(app)/jobs/[id].tsx` shows a "CLIENT" eyebrow + avatar (40dp) + name (if non-null) + email inline after the job description.
- Identity section is always rendered when job data is loaded — no status gate (D-02).
- On `getUser` error, section is silently omitted (not shown, not errored).
- Imports in `apps/mobile/lib/api.ts` and mobile screen use `@local/types` (correct for mobile).

## Truths

- D-02: Section is always shown when job loads — no conditional by status.
- D-09: `name` is omitted from UI when null; email is always shown.
- D-11: `getUser` signature is `(token: string, userId: string): Promise<PublicUserDto>`. Delegates to `parseResponse<PublicUserDto>(res)`.
- Mobile uses `@local/types` for all type imports (not `@/lib/types`).
- `Avatar.Text` uses a `label` prop (string), not `initials`. `Avatar.Image` uses `source={{ uri: avatarUrl }}`.
- The "CLIENT" eyebrow uses `metaLabel` style: `{ fontSize: 11, fontWeight: '700', letterSpacing: 0.6, textTransform: 'uppercase', color: '#94a3b8' }`.
- Identity section is separated from the description block above it with a `<Divider />` from `react-native-paper`.

<action>
Create `apps/mobile/components/AvatarInitials.tsx`:

```typescript
import { Avatar } from 'react-native-paper'

interface AvatarInitialsProps {
  name: string | null
  email: string
  avatarUrl: string | null
  size: number
}

export default function AvatarInitials({ name, email, avatarUrl, size }: AvatarInitialsProps) {
  const initials = name
    ? name.trim().split(/\s+/).map(w => w[0]).slice(0, 2).join('').toUpperCase()
    : email[0].toUpperCase()

  if (avatarUrl) {
    return <Avatar.Image size={size} source={{ uri: avatarUrl }} />
  }

  return <Avatar.Text size={size} label={initials} />
}
```

`size` is a numeric dp value: 40 for inline identity sections. The `Avatar.Text` background color will follow the app's Paper theme; if the theme default differs from `surface-900`, add a `style={{ backgroundColor: '#0f172a' }}` prop on `Avatar.Text` to match the design spec.
</action>

**Step 3 — Add useEffect** (after the loadJob useEffect):
```typescript
useEffect(() => {
  if (!job || !token) return

  const fetchClient = async () => {
    setIsClientLoading(true)
    try {
      const user = await getUser(token, job.clientId)
      setClientUser(user)
    } catch {
      // silently omit section on error
    } finally {
      setIsClientLoading(false)
    }
  }

  fetchClient()
}, [job, token])
```
