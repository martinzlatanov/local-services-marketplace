# Plan 13-02 — API: GET /api/users/[id]

## Goal

Create the `GET /api/users/[id]` endpoint that returns a `PublicUserDto` for a given user ID. This is the single new backend endpoint for Phase 13. All web and mobile UI plans (13-03, 13-04) depend on this endpoint being available.

## Must-Haves

- `apps/web/app/api/users/[id]/route.ts` exists and exports a `GET` handler.
- Unauthenticated requests return HTTP 401 with `{ errors: { auth: 'unauthorized' } }`.
- Non-numeric or NaN `id` param returns HTTP 400 with `{ errors: { id: 'invalid' } }`.
- Unknown numeric ID returns HTTP 404 with `{ errors: { user: 'not_found' } }`.
- Happy path returns HTTP 200 with `{ data: PublicUserDto }` — fields: `id` (string), `email`, `name`, `avatarUrl`, `role`, `createdAt`.
- `passwordHash` is NEVER present in the response — explicit DTO projection only, no spread of the row object.
- Imports use `@/lib/types` (NOT `@local/types` — that causes build errors in web routes).

## Truths

- D-04: Auth is required. Use `getAuthenticatedUser(req)` from `@/lib/auth` verbatim.
- D-10: DTO shape is locked. Build the DTO object explicitly, field by field.
- `users.id` is a serial integer in the DB; the URL param is a string. Always `parseInt(id, 10)` + `isNaN` guard.
- `row.name` and `row.avatarUrl` may be `undefined` from Drizzle (nullable columns default to `undefined` not `null`). Use `?? null` when assigning to DTO fields.
- All API routes follow static top-level imports (not lazy imports) — see `jobs/[id]/route.ts`.

Create `apps/web/app/api/users/[id]/route.ts` with the following implementation:

```typescript
import { NextResponse } from 'next/server'
import { db } from '@/lib/db/client'
import { users } from '@/lib/db/schema'
import { getAuthenticatedUser } from '@/lib/auth'
import { PublicUserDto, ApiSuccessResponse, Role } from '@/lib/types'
import { eq } from 'drizzle-orm'

export async function GET(req: Request, { params }: { params: Promise<{ id: string }> }) {
  const user = await getAuthenticatedUser(req)
  if (!user) return NextResponse.json({ errors: { auth: 'unauthorized' } }, { status: 401 })

  const { id } = await params
  const userId = parseInt(id, 10)
  if (isNaN(userId)) {
    return NextResponse.json({ errors: { id: 'invalid' } }, { status: 400 })
  }

  const [row] = await db.select().from(users).where(eq(users.id, userId)).limit(1)
  if (!row) {
    return NextResponse.json({ errors: { user: 'not_found' } }, { status: 404 })
  }

  const dto: PublicUserDto = {
    id: String(row.id),
    email: row.email,
    name: row.name ?? null,
    avatarUrl: row.avatarUrl ?? null,
    role: row.role as Role,
    createdAt: row.createdAt.toISOString(),
  }

  return NextResponse.json({ data: dto } as ApiSuccessResponse<PublicUserDto>)
}
```

Key guards:
- Import path is `@/lib/types` (not `@local/types`)
- DTO is built field-by-field — never spread `row`
- `row.name ?? null` and `row.avatarUrl ?? null` handle Drizzle's nullable-as-undefined convention
