# Plan 13-03 — Web UI: Identity Components and Provider Profile

## Goal

Build the web-side identity UI: a shared `AvatarInitials` component, inline provider identity in `JobDetailCard`, a new public provider profile page at `/providers/[id]`, and deletion of the stale partial implementation at `/dashboard/provider/[id]`.

## Must-Haves

- `apps/web/components/ui/AvatarInitials.tsx` exists; accepts `{ name, email, avatarUrl, size: 'sm' | 'lg' }` props.
- `JobDetailCard.tsx` shows provider avatar + name + email + "View profile" link when `job.providerId !== null`. Condition is NOT status-gated.
- `apps/web/app/providers/[id]/page.tsx` exists with: profile header (AvatarInitials lg + name/email/member-since), average star rating section, approved reviews list, empty state ("No reviews yet."), error state ("Couldn't load this profile. Refresh to try again."), loading skeleton.
- Old route `apps/web/app/dashboard/provider/[id]/page.tsx` is deleted.
- Stars use `fill-accent-500 stroke-accent-500` (amber) — not `fill-brand-500`.
- When `name` is null: avatar shows email initial, name field is omitted (not replaced with placeholder).
- "View profile" link in JobDetailCard: `text-[12px] text-surface-900 underline underline-offset-2 hover:text-surface-600 transition-colors`.

## Truths

- D-01: Condition is `job.providerId !== null`, not `job.status === 'ACCEPTED'`. A job with status IN_PROGRESS or COMPLETED can also have a providerId.
- D-06: Initials logic — if name: first letter of each word (max 2, uppercase). If name null: first letter of email, uppercase.
- D-07: Provider profile shows "No reviews yet." when reviews array is empty.
- D-09: `name` field is omitted from UI when null — do not render a dash or "Unknown" placeholder.
- Average rating formula: `(averageRatings.communication + averageRatings.quality + averageRatings.punctuality) / 3`. Show "No rating yet" when all three are 0 or reviews array is empty.
- Avatar colors: `bg-surface-900` (`#0f172a`), text `text-surface-0` (`#ffffff`).
- Inline skeleton (loading state): avatar `w-8 h-8 rounded-full bg-surface-100 animate-pulse` + two line placeholders.
- Profile page layout: `max-w-3xl mx-auto px-4 py-8`; cards use `bg-surface-0 border border-surface-200 rounded-[var(--radius-card)]`.

```typescript
interface AvatarInitialsProps {
  name: string | null
  email: string
  avatarUrl: string | null
  size: 'sm' | 'lg'
}

export default function AvatarInitials({ name, email, avatarUrl, size }: AvatarInitialsProps) {
  const initials = name
    ? name.trim().split(/\s+/).map(w => w[0]).slice(0, 2).join('').toUpperCase()
    : email[0].toUpperCase()

  const dimensions = size === 'sm' ? 'w-8 h-8 text-[12px]' : 'w-20 h-20 text-[28px]'

  if (avatarUrl) {
    return <img src={avatarUrl} className={`${dimensions} rounded-full object-cover`} alt="" aria-hidden="true" />
  }

  return (
    <div className={`${dimensions} rounded-full bg-surface-900 text-surface-0 flex items-center justify-center font-semibold select-none`}>
      {initials}
    </div>
  )
}
```

No `'use client'` directive needed (no hooks). No state. Pure props-in, JSX-out.

```typescript
useEffect(() => {
  const fetchData = async () => {
    setIsLoading(true)
    setError(null)
    try {
      const [userRes, reviewsRes] = await Promise.all([
        fetch(`/api/users/${providerId}`, { credentials: 'include' }),
        fetch(`/api/reviews?userId=${providerId}&approved=true`, { credentials: 'include' }),
      ])
      if (!userRes.ok || !reviewsRes.ok) throw new Error('fetch_failed')
      const userData = await userRes.json()
      const reviewsData = await reviewsRes.json()
      setProviderUser(userData.data)
      setReviews(reviewsData.data?.reviews || [])
      setAverageRatings(reviewsData.data?.averageRatings || {})
    } catch {
      setError("Couldn't load this profile. Refresh to try again.")
    } finally {
      setIsLoading(false)
    }
  }
  fetchData()
}, [providerId])
```
