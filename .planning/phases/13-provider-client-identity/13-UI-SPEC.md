## Component Inventory

### 1 — Avatar Initials Circle (new, shared)

A reusable presentational component used in both inline identity sections and the profile page header.

**When `avatarUrl` is null (initials mode):**
- Circular, surface-900 background, white initials text
- Initials derived from `name` (first letter of each word, max 2 chars) or from `email` (first char, uppercased) when name is null
- Web: `rounded-full bg-surface-900 text-surface-0 flex items-center justify-center font-semibold select-none`

**When `avatarUrl` is set (image mode):**
- Same circular bounds, `<img>` or `<Image>` with `object-cover`, `rounded-full`

**Sizes:**
| Context | Size | Font size |
|---------|------|-----------|
| Inline web (job card) | 32×32px | 12px / weight 600 |
| Inline mobile (job detail) | 40×40dp | 14sp / weight 600 |
| Profile page header | 80×80px | 28px / weight 600 (component-local override) |

### 2 — Inline Identity Section — Web (inside existing job card)

Shown within the existing `JobCard` / `JobDetailCard` component when `job.providerId` is non-null.

**Structure:**
```
[eyebrow label: "PROVIDER"]
[Avatar 32px] [Name if non-null — 14px/400/surface-800]
              [Email — 12px/400/surface-500]
              ["View profile" link → /providers/[id]]
```

**Loading state:** While `GET /api/users/[providerId]` is in flight, render a skeleton placeholder:
- Avatar placeholder: `w-8 h-8 rounded-full bg-surface-100 animate-pulse`
- Name/email lines: two div rectangles `h-3 rounded bg-surface-100 animate-pulse` (widths: 80px and 120px)
- No spinner. Resolves silently.

**Separator:** 1px `border-t border-surface-200` above the identity section, `mt-3 pt-3` spacing.

**"View profile" link:** `text-[12px] text-surface-900 underline underline-offset-2 hover:text-surface-600 transition-colors`. Navigates to `/providers/[id]`.

### 3 — Inline Identity Section — Mobile (inside existing job detail screen)

Shown at the bottom of the job detail screen, always rendered when job data is loaded.

**Structure (label/value rows using React Native Paper):**
```
Section header: "CLIENT" (eyebrow style — Text variant="labelSmall", uppercase, surface-500)
Row: [Avatar 40dp] [Name (if non-null) — Text variant="bodyMedium" weight 600]
                   [Email — Text variant="bodySmall" color theme.colors.onSurfaceVariant]
```

**Loading state:** While `getUser(token, clientId)` is in flight, render:
- Avatar: `View` with `width: 40, height: 40, borderRadius: 20, backgroundColor: theme.colors.surfaceVariant`
- Name/email: two `View` rectangles with `backgroundColor: theme.colors.surfaceVariant` and pulsing opacity via Animated API (matching mobile existing loading patterns)
- No spinner.

**Separator:** `Divider` component from React Native Paper above the client section.

### 4 — Provider Profile Page (`/providers/[id]`)

New page at `apps/web/app/providers/[id]/page.tsx`.

**Page layout:** Single column, `max-w-3xl mx-auto px-4 py-8`, page background `bg-surface-50`.

**Profile header card:**
- `bg-surface-0 border border-surface-200 rounded-[var(--radius-card)] p-6 shadow-[var(--shadow-card)]`
- Layout: `flex items-start gap-4`
- Left: Avatar circle 80×80px
- Right (stacked):
  - Name (if non-null): `text-[20px] font-semibold text-surface-800 tracking-[-0.5px]`
  - Email: `text-[14px] text-surface-500`
  - Member since: `text-[12px] text-surface-400` — format: `Joined {Month YYYY}` (e.g. "Joined January 2025")
  - Star rating: numeric average + Star icons (see rating display spec below)

**Rating display — profile header:**
- If reviews exist: render average to one decimal (e.g. "4.3") + 5 Star icons (lucide `Star`, 16px)
  - Filled stars: `fill-accent-500 stroke-accent-500`
  - Partial/empty stars: `stroke-surface-300 fill-none`
  - Average computed from `clientCommunication`, `clientQuality`, `clientPunctuality` fields on reviews where `reviewType === 'client'`
  - Formula: mean of all three fields across all qualifying reviews
- If no reviews or zero qualifying reviews: display string `No rating yet` in `text-[14px] text-surface-400`

**Reviews list:**
- Section eyebrow: "REVIEWS" (`.eyebrow` utility class)
- Reviews stacked as surface-0 cards, `mt-4 flex flex-col gap-3`
- Each review card: `bg-surface-0 border border-surface-200 rounded-[var(--radius-card)] p-4`
  - Top row: reviewer email (`text-[12px] text-surface-700`) + date (`text-[11px] text-surface-400`) — space-between
  - Star row: 5 Star icons at 14px using same fill/stroke contract as header
  - Review text: `text-[14px] text-surface-600 mt-2 leading-relaxed`
  - Photo (if `photoUrl` set): `<img>` with `w-full max-h-48 object-cover rounded-lg mt-3`

**Empty reviews state:**
- Profile header renders normally (avatar, email, name, member since, "No rating yet")
- In place of the reviews list: centered empty-state block
  ```
  [eyebrow: "REVIEWS"]
  <p class="text-[14px] text-surface-400 mt-4">No reviews yet.</p>
  ```
- No illustration, no secondary CTA — phase is read-only

**Loading state (whole page):** While fetching, render skeleton cards using `animate-pulse` divs matching the final layout dimensions. No spinner.

**Error state (whole page):** If `/api/users/[id]` returns 4xx/5xx:
- Render: `"Couldn't load this profile. Refresh to try again."`
- Style: `text-[14px] text-surface-500 text-center mt-16`
