  <files>apps/mobile/lib/api.ts, apps/mobile/hooks/useJobsWebSocket.ts</files>
  <action>
    - Create apps/mobile/lib/api.ts with functions:
      - `getJobs(token, cityArea)` -> GET `${API_BASE}/api/jobs?cityArea=${encodeURIComponent(cityArea)}` and returns `JobDto[]`.
      - `getJob(token, id)` -> GET `${API_BASE}/api/jobs/${id}` and returns `JobDto`.
      - `acceptJob(token, id, version)` -> POST `${API_BASE}/api/jobs/${id}/accept` with JSON body `{ version }` and returns the updated `JobDto`.
      - Each function should throw the parsed `ApiErrorResponse` on non-2xx and include `Authorization: Bearer ${token}` header.
    - Create apps/mobile/hooks/useJobsWebSocket.ts that:
      - Accepts `{ token, onJobUpdated }` and manages connection state.
      - Builds WS URL by replacing `http://` with `ws://` from API_BASE and appending `?token=${token}`.
      - On message, parse JSON, check `type === 'JOB_UPDATED'`, and call `onJobUpdated(payload)`.
      - Reconnects after 3s on close and calls an optional `onReconnect` callback to trigger a refetch.
  </action>
  <acceptance_criteria>
    - apps/mobile/lib/api.ts exports `getJobs`, `getJob`, and `acceptJob` and uses the Bearer token header.
    - apps/mobile/hooks/useJobsWebSocket.ts references `JOB_UPDATED` and uses `ws://` URL derived from API_BASE.
  </acceptance_criteria>
  <verify>
    <automated>npm run typecheck --workspace=apps/mobile</automated>
  </verify>
