# Plan 03: Frontend Review UI — Forms & Profile Display

## Objective

Implement client-facing review UI components and integrate them into job detail and provider profile pages. Users can submit reviews after completing jobs; profiles display approved reviews with aggregate ratings.

**Purpose:** Enable users to submit and view reviews through the UI.

**Output:** ReviewForm and ReviewDisplay components + page integrations.

## Context

Builds on Wave 2 (API endpoints). Review data fetched from `/api/reviews` and `/api/reviews?userId=X&approved=true`.

Form framework likely React Hook Form + Zod (check project setup). Styling via Tailwind or existing CSS framework.

## Tasks

### Task 1: Create ReviewForm component (client and provider variants)

**File:** `apps/web/components/ReviewForm.tsx`

Create a reusable ReviewForm component that adapts to review direction (client→provider or provider→client).

**Props:**
- `jobId: number`
- `reviewType: 'client' | 'provider'`
- `onSuccess: () => void` (callback after successful submission)
- `reviewerUserId: number`
- `revieweeUserId: number`

**UI Structure:**
1. **Heading:** "Rate [name]" or "Review Your Experience"
2. **Rating section:** For each category (determined by reviewType):
   - Label: e.g., "Communication" (client) or "Payment Reliability" (provider)
   - Stars: 1–5 interactive star rating (hover preview, click select)
   - Visual feedback: Show selected number + label (e.g., "4 - Good")
3. **Text input:** textarea, max 1000 chars, char counter ("250/1000")
4. **Photo upload:**
   - Drag-drop zone or file input
   - Accept image/* (JPEG, PNG, WebP)
   - Validate < 5 MB
   - Show file size
   - Preview thumbnail inline
   - Upload to Vercel Blob, get URL, include in form data
5. **Buttons:**
   - Submit button (disabled until all ratings filled)
   - Cancel button

**Behavior:**
- On submit: POST /api/reviews with {jobId, [clientRating | providerRating], text, photoUrl}
- Loading state: Disable submit, show spinner
- Success: Call onSuccess() callback
- Error: Display error message (e.g., "409: Already reviewed")
- Duplicate detection: Disable form with message

Use form library consistent with project (React Hook Form + Zod). Match component styling to existing UI.

**Verification:**
```bash

grep "POST.*api/reviews" apps/web/components/ReviewForm.tsx
```

### Task 2: Create ReviewDisplay component for profile view

**File:** `apps/web/components/ReviewDisplay.tsx`

Create ReviewDisplay component to render approved reviews on a user's profile.

**Props:**
- `userId: number` (profile owner being reviewed)
- `reviews: ReviewDTO[]` (fetched from API)
- `averageRatings: {categoryName: number}` (e.g., {communication: 4.2, quality: 4.5, punctuality: 4.0})
- `isLoading: boolean`
- `reviewType: 'client' | 'provider'` (so labels match)

**UI Structure:**
1. **Header section:**
   - Title: "Client Reviews" or "Provider Reviews"
   - Average rating: Large star icon + score (e.g., "4.3 out of 5")
   - Per-category breakdown: "Communication: 4.2 | Quality: 4.5 | ..."
   - Review count: "Based on {count} reviews"
2. **Reviews list:**
   - For each review:
     - Reviewer name + date (e.g., "John D. • 3 weeks ago")
     - Stars (5-star display, not interactive)
     - Rating text: e.g., "Communication: 5, Quality: 4, Punctuality: 5"
     - Review text (truncate at 300 chars with "Read more" link if long)
     - Photo: Thumbnail; click → full-size modal
3. **Empty state:** "No reviews yet" if reviews.length === 0
4. **Loading state:** Skeleton/spinner if isLoading

**Behavior:**
- Render only approved reviews (API returns only approvedAt IS NOT NULL)
- Photo modal: Click photo → display full-size with close button
- No edit/delete controls (reviews immutable once approved)

### Task 3: Integrate ReviewForm into job detail page

**File:** `apps/web/app/(dashboard)/jobs/[id]/page.tsx`

Integrate ReviewForm to allow users to submit a review after job completion.

**Changes:**
1. Import ReviewForm component
2. Fetch job details (already exists)
3. Determine if current user is client or provider (set reviewType)
4. Check if review already submitted:
   - GET /api/reviews?jobId={id}
   - Set hasReviewed if any review exists with reviewerId === authenticated user
5. Conditional render:
   - If status !== 'COMPLETED': hide form, show "Leave a review after job completes"
   - If hasReviewed: show "You've reviewed this job" badge
   - If !hasReviewed && status === 'COMPLETED': show ReviewForm
6. onSuccess handler:
   - Refresh reviews list
   - Show toast: "Review submitted! Pending admin approval."
   - Optionally disable form or show success state

Position review section below job details, above timeline/chat if present.

Do not modify job detail fetching; only add conditional rendering.

## Threat Model

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User→ReviewForm | Untrusted: text content, photo upload |
| Browser→API (photo URL) | Partially trusted: URL origin verified to be Vercel Blob |
| Profile page→Review data | Trusted: backend enforces approvedAt IS NOT NULL |

### STRIDE Register

| Threat ID | Category | Component | Mitigation |
|-----------|----------|-----------|-----------|
| T-11-10 | Information Disclosure | review text | Client-side XSS: sanitize text output (use React innerText or trusted HTML serializer) |
| T-11-11 | Tampering | photo upload | File size validated client + API-side; content-type checked |
| T-11-12 | Denial of Service | text field | Max 1000 chars enforced; low risk |
| T-11-13 | Information Disclosure | pending reviews | ReviewForm hides pending; only author sees pending review details |
