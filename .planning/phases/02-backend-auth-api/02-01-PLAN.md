<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shared types -> route handlers | Auth request and validation contracts are consumed by runtime code; malformed shapes must not leak into implementation assumptions |
| route handlers -> database | User creation crosses into persistent storage and must preserve unique email and password-hash-only storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | T | packages/types/src/index.ts | mitigate | Encode the validation error contract and auth DTOs once in the shared barrel so route handlers do not invent incompatible local shapes |
| T-02-02 | T | apps/web/lib/db/schema.ts | mitigate | Enforce unique email and role constraints in the Drizzle schema so duplicate registration and role tampering fail before persistence |
| T-02-03 | I | apps/web/lib/db/client.ts | mitigate | Keep only database connection setup in the client helper and store password hashes, never raw passwords, in the schema or migration |
</threat_model>
