Create `tsconfig.base.json` at the repo root with EXACTLY this content (per D-10, D-11 and research Pattern 2 â€” these settings match expo/tsconfig.base for Metro compatibility AND Next.js 16 defaults):

```json
{
  "compilerOptions": {
    "target": "ESNext",
    "lib": ["DOM", "DOM.Iterable", "ESNext"],
    "moduleResolution": "bundler",
    "module": "preserve",
    "jsx": "react-jsx",
    "strict": true,
    "skipLibCheck": true,
    "noEmit": true,
    "esModuleInterop": true,
    "resolveJsonModule": true,
    "isolatedModules": true
  }
}
```

Rationale (do not deviate):
- `moduleResolution: bundler` + `module: preserve` is the canonical TS 5.4+ pair for bundler environments (Turbopack and Metro). `nodenext` would break Metro.
- `jsx: react-jsx` is the Expo default; `apps/web` will override to `preserve` in plan 01-03.
- `strict: true` is non-negotiable per D-10.
- `noEmit: true` because we never compile from this config (apps emit through their bundlers; types package exports raw source).
  </action>
