<task type="auto">
  <name>Task 1: Create vercel.json configuration</name>
  <files>vercel.json</files>
  <action>
Create a `vercel.json` file at the project root to configure Vercel deployment for the monorepo.

**Implementation:**
```json
{
  "version": 2,
  "name": "local-services-marketplace",
  "builds": [
    {
      "src": "apps/web/package.json",
      "use": "@vercel/next"
    }
  ],
  "routes": [
    {
      "src": "/(.*)",
      "dest": "/apps/web/$1"
    }
  ]
}
```

**Alternative (simpler):** If using Vercel's monorepo support with `projects` config:
```json
{
  "version": 2,
  "name": "local-services-marketplace",
  "projects": {
    "apps/web": {}
  }
}
```

**Recommendation:** Use the simpler approach — just ensure Vercel is connected to the GitHub repo and set the root directory to `apps/web` in Vercel project settings. Create a minimal `vercel.json`:
```json
{
  "version": 2,
  "name": "local-services-marketplace"
}
```

**Why:** This enables Vercel to recognize the project. The actual build configuration comes from `apps/web/package.json` scripts.

Also create `VERCEL-DEPLOYMENT.md` with step-by-step deployment instructions:
```markdown
# Vercel Deployment Guide

## Prerequisites
1. Vercel account connected to GitHub
2. Neon production database provisioned

## Steps
1. Run `npm install -g vercel`
2. Run `vercel login`
3. Run `vercel --prod` from project root
4. In Vercel dashboard: Settings → Environment Variables → Add all vars from .env.production
5. Redeploy if needed: `vercel --prod`

## Environment Variables
- DATABASE_URL: Neon production connection string
- JWT_SECRET: Strong secret key
- NEXT_PUBLIC_WS_URL: (Optional) WebSocket URL
```

**Why:** Documents the deployment process and provides a checklist of environment variables needed in Vercel.
